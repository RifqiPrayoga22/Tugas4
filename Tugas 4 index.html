Polimorfisme (Polymorphism)

📖 Definisi

Poly = banyak, morph = bentuk. Objek dari berbagai subclass diperlakukan sebagai objek dari superclass yang sama.

Runtime polymorphismMethod overriding

✅ Manfaat

  • Fleksibilitas – kode bekerja dengan berbagai tipe
  • Extensibility – mudah tambah tipe baru
  • Reusability – method umum di superclass
  • Maintainability – perubahan mudah dikelola

Dart mendukung runtime polymorphism via method overriding. Compile-time (overloading) tidak didukung.

🎯 Polimorfisme Dasar Buka di DartPad 
class Bentuk {
  void gambar() => print('Menggambar bentuk');
}
class Lingkaran extends Bentuk {
  @override
  void gambar() => print('Menggambar lingkaran ◯');
}
class Persegi extends Bentuk {
  @override
  void gambar() => print('Menggambar persegi ◱');
}
class Segitiga extends Bentuk {
  @override
  void gambar() => print('Menggambar segitiga △');
}
void main() {
  List<Bentuk> bentuk = [Lingkaran(), Persegi(), Segitiga()];
  for (var b in bentuk) {
    b.gambar();
  }
}
▶ Expected Output
Menggambar lingkaran
Menggambar persegi
Menggambar segitiga

Abstraksi — Abstract Class

📖 Konsep

Menyembunyikan detail implementasi. Fokus pada "apa" bukan "bagaimana".

🛠 Aturan

  • Tidak bisa diinstansiasi langsung
  • Method abstract = tanpa body, wajib di-override
  • Method concrete = sudah ada implementasi
🎯 Abstract Class Buka di DartPad 
abstract class Kendaraan {
  String? merk;
  void start();          // abstract
  String getInfo();      // abstract
  void stop() { print('$merk berhenti'); }  // concrete
}
class Mobil extends Kendaraan {
  Mobil({required String merk}) { this.merk = merk; }
  @override void start() => print('$merk: Mesin mobil menyala');
  @override String getInfo() => 'Mobil merk $merk';
}
class SepedaMotor extends Kendaraan {
  SepedaMotor({required String merk}) { this.merk = merk; }
  @override void start() => print('$merk: Mesin motor nyala');
  @override String getInfo() => 'Sepeda motor merk $merk';
}
void main() {
  List<Kendaraan> garasi = [Mobil(merk: 'Toyota'), SepedaMotor(merk: 'Honda')];
  for (var k in garasi) { print(k.getInfo()); k.start(); k.stop(); print('---'); }
}
▶ Expected Output
Mobil merk Toyota
Toyota: Mesin mobil menyala
Toyota berhenti
---
Sepeda motor merk Honda
Honda: Mesin motor nyala
Honda berhenti
---

🎯 Interface & Multiple Implements Buka di DartPad 
interface class Flyable { void terbang(); }
interface class Swimmable { void berenang(); }
class Burung implements Flyable {
  @override void terbang() => print('Burung terbang melintas tinggi');
}
class Pesawat implements Flyable {
  @override void terbang() => print('Pesawat terbang cepat');
}
class Itik implements Flyable, Swimmable {
  @override void terbang() => print('Itik terbang rendah');
  @override void berenang() => print('Itik berenang di kolam');
}
void main() {
  List<Flyable> terbang = [Burung(), Pesawat(), Itik()];
  print('=== Semua yang bisa terbang ===');
  for (var f in terbang) f.terbang();
  print('\n=== Yang bisa berenang ===');
  Itik().berenang();
}
▶ Expected Output

Static Members & Methods

ℹ️ Konsep

Static member dimiliki class itu sendiri, bukan objek. Diakses via nama class tanpa membuat objek. Berguna untuk utility class dan konstanta.

🎯 Static Members & Methods Buka di DartPad 
class MathUtils {
  static const double pi = 3.14159;
  static int add(int x, int y) => x + y;
  static int multiply(int x, int y) => x * y;
  static double lingkaranLuas(double r) => pi * r * r;
}
void main() {
  print('Pi = ${MathUtils.pi}');
  print('10 + 20 = ${MathUtils.add(10, 20)}');
  print('6 x 7 = ${MathUtils.multiply(6, 7)}');
  print('Luas r=5 = ${MathUtils.lingkaranLuas(5).toStringAsFixed(2)}');
}
▶ Expected Output
Pi = 3.14159
10 + 20 = 30
6 x 7 = 42
Luas r=5 = 78.54

Studi Kasus: Sistem Pembayaran

Penggunaan nyata abstract class + interface dalam sistem multi-metode pembayaran.

🎯 Studi Kasus: Sistem Pembayaran Buka di DartPad 
abstract class MetodePembayaran {
  void bayar(double jumlah);
  String getNama();
}
class KartuKredit implements MetodePembayaran {
  String noKartu; KartuKredit(this.noKartu);
  @override
  void bayar(double j) => print('Bayar Rp${j.toStringAsFixed(0)} Kartu Kredit (****${noKartu.substring(noKartu.length-4)})');
  @override String getNama() => 'Kartu Kredit';
}
class EWallet implements MetodePembayaran {
  String provider; EWallet(this.provider);
  @override
  void bayar(double j) => print('Bayar Rp${j.toStringAsFixed(0)} EWallet $provider');
  @override String getNama() => 'E-Wallet $provider';
}
class Tunai implements MetodePembayaran {
  @override
  void bayar(double j) => print('Bayar Rp${j.toStringAsFixed(0)} Tunai');
  @override String getNama() => 'Tunai';
}

📝 Tugas 1 + BONUS 1 — Sistem Pekerja & Polimorfisme

Abstract class Pekerja → class Programmer, Dokter, Guru → List<Pekerja> demonstrasi polimorfisme.

🎯 Bonus 1 — Sistem Pekerja & Polimorfisme Buka di DartPad 
abstract class Pekerja {
  String nama; int tahunPengalaman;
  Pekerja(this.nama, this.tahunPengalaman);
  void bekerja();  // abstract
  void perkenalkanDiri() {  // concrete
    print('Hai, saya $nama dengan $tahunPengalaman tahun pengalaman.');
  }
}
class Programmer extends Pekerja {
  String bahasa;
  Programmer(String nama, int tahun, this.bahasa) : super(nama, tahun);
  @override void bekerja() => print('[Programmer] $nama menulis kode $bahasa...');
}
class Dokter extends Pekerja {
  String spesialisasi;
  Dokter(String nama, int tahun, this.spesialisasi) : super(nama, tahun);
  @override void bekerja() => print('[Dokter] $nama memeriksa pasien $spesialisasi...');
}
class Guru extends Pekerja {
  String mataPelajaran;
  Guru(String nama, int tahun, this.mataPelajaran) : super(nama, tahun);
  @override void bekerja() => print('[Guru] $nama mengajar $mataPelajaran...');
}
void main() {
  List<Pekerja> tim = [
    Programmer('Ali', 5, 'Dart'),
    Dokter('Sana', 10, 'Jantung'),
    Guru('Budi', 8, 'Matematika'),
  ];
  for (var p in tim) { p.perkenalkanDiri(); p.bekerja(); print('--'); }
}

📝 BONUS 2 — Class Tablet (Multiple Inheritance)

Tablet implements Komputer, Kamera, dan Telepon sekaligus — multiple interface.

🎯 Bonus 2 — Tablet Multiple Inheritance Buka di DartPad 
interface class Komputer { void jalankanApp(String app); void browse(String url); }
interface class Kamera   { void ambilFoto(int mp); void rekamVideo(int fps); }
interface class Telepon  { void telepon(String nomor); void kirimSms(String pesan); }

class Tablet implements Komputer, Kamera, Telepon {

📝 BONUS 3 — MathUtils Static Methods Lengkap

Faktorial, cek prima, pembulatan, acakInt, acakDouble, rataRata, dan median — semuanya static method.

🎯 Bonus 3 — MathUtils Static Methods Lengkap Buka di DartPad 
import 'dart:math';
class MathUtils {
  static int faktorial(int n) { if (n<=1) return 1; return n * faktorial(n-1); }
  static bool adalahPrima(int n) {
    if (n<2) return false;
    for (int i=2; i<=sqrt(n.toDouble()).toInt(); i++)
      if (n%i==0) return false;
    return true;
  }
  static double bulatkan(double n, int d) {
    double f = pow(10,d).toDouble(); return (n*f).round()/f;
  }
  static int    acakInt(int min, int max)       => min + Random().nextInt(max-min+1);
  static double acakDouble(double min, double max) => min + Random().nextDouble()*(max-min);
  static double rataRata(List<num> a) => a.reduce((x,y)=>x+y) / a.length;
  static double median(List<num> a) {
    List<num> s = List.from(a)..sort(); int m = s.length~/2;
    return s.length%2==0 ? (s[m-1]+s[m])/2 : s[m].toDouble();
  }
}

Ringkasan Sesi 4

🎯 Poin Utama

  1. Polimorfisme — Objek child dapat diperlakukan sebagai parent. Runtime polymorphism via method overriding. Gunakan is untuk cek tipe dan as untuk casting.
  2. Abstract Class — Template class yang tidak bisa diinstansiasi langsung. Punya method abstract (wajib di-override) dan concrete (sudah ada body).
  3. Interface — Keyword interface class sejak Dart 3.0. Semua method harus di-override. Mendukung multiple implements.
  4. Multiple Inheritance — Dart tidak support multiple extends, tapi support multiple implements.
  5. Static Members — Milik class, bukan objek. Diakses via nama class. Cocok untuk utility dan konstanta.
